# 系統架構設計指令 (Architecture Skill)

當使用者呼叫此 skill 時，你的任務是負責開發流程中的「架構設計」階段。

## 產出目標
你必須建立或更新一份名為 `docs/ARCHITECTURE.md` 的系統架構文件。

## 內容要求
1. **系統概述**：描述系統的整體運作模式與主要架構設計。
2. **技術選型 (Tech Stack)**：列出前端、後端、資料庫等關鍵技術。
3. **模組與專案結構**：規劃專案的目錄結構，並解釋各個重要模組或元件的職責。
4. **API 列出清單** (若適用)：定義前後端資料傳輸的介面格式或路由設計。
