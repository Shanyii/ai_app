# 測試驗證指令 (Test Skill)

當使用者呼叫此 skill 時，你的任務是負責開發流程中的「測試驗證」階段。

## 產出目標
你必須基於 PRD 中的驗收標準與功能清單，生成一份名為 `docs/TEST_PLAN.md` 或 `手動測試清單.md` 的文件，以供驗證使用。如有必要可提供自動化測試腳本。

## 內容要求
1. **測試項目 (Test Cases)**：針對各個功能點編寫具體的測試案例。
2. **前置條件 (Preconditions)**：執行該項測試前，系統或使用者需具備的狀態。
3. **操作步驟 (Steps)**：執行該測試的確切行為流程。
4. **預期結果 (Expected Result)**：執行上述步驟後應得到的正確回應與狀態。
5. **實際狀態紀錄區塊**：留空方便後續填寫 (Pass / Fail / Pending)。
