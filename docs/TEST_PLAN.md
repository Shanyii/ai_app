# 系統測試計畫 (TEST PLAN) - 線上宮廟與儲值系統

這份測試計畫基於先前定義的 PRD，將這套系統的核心功能拆分為具體的測試用例。請測試人員依照每個步驟手動點擊執行，並在「實際狀態」填寫驗證結果。

## 測試案例一：會員系統 (註冊與登入)
* **前置條件**：無。確保瀏覽器處於未登入狀態，且伺服器正在掛載中。
* **操作步驟**：
  1. 點擊導覽列的「會員登入」。
  2. 於登入視窗下方點擊「沒有帳號？立即註冊」。
  3. 輸入測試 Email (如 `test@example.com`) 與任意密碼，點擊「確認註冊」。
  4. 註冊成功後，若您的系統沒有設定為自動登入，請嘗試使用同一個帳號進行「會員登入」。
* **預期結果**：
  - 註冊時系統會提示「註冊成功」。
  - 登入後，右上角的按鈕將轉變為「切換帳號」或「登出」，並且旁邊會出現有您信箱帳號字首的問候語。
* **實際狀態**：[ ] Pass / [ ] Fail / [ ] Pending


## 測試案例二：點擊求籤與寫入紀錄
* **前置條件**：確保已成功登入一組測試帳號。
* **操作步驟**：
  1. 於首頁大廳，點擊畫面中央的「求籤」靈籤筒。
  2. 觀察搖晃動畫的視覺體驗。
  3. 等待數秒後，下方會彈出「求得第 N 首籤詩」的提示與紀錄字樣。
* **預期結果**：
  - 籤筒必須有物理搖晃或位移效果。
  - 提示字樣會確實告知您所求得的隨機籤號（介於 1-60 之間）。
  - 後台系統會順利地將這次的紀錄打入 SQLite 資料庫中 (無報錯發生)。
* **實際狀態**：[ ] Pass / [ ] Fail / [ ] Pending


## 測試案例三：體驗香火儲值金流
* **前置條件**：已經登入測試帳號。請注意右上角「香火庫存」的初始數字（新帳號應為 `$0`）。
* **操作步驟**：
  1. 點擊導覽列最右方或畫面中的「香火儲值」按鈕。
  2. 於彈窗中點擊固定的 `NT$ 500` 方案按鈕（或嘗試手動輸入欄位）。
  3. 點擊最下方的「確認儲值」提交金流訂單。
* **預期結果**：
  - 點擊後，系統會自動彈出「感謝隨喜」的成功提示。
  - 上方導覽列的「香火庫存」餘額數字會即刻增加，例如從 `$0` 變為 `$500`。
* **實際狀態**：[ ] Pass / [ ] Fail / [ ] Pending


## 測試案例四：雙分頁歷史明細切換
* **前置條件**：確保該帳號日前已至少執行過一次「求籤」(案例二) 與一次「儲值」(案例三)。
* **操作步驟**：
  1. 點擊導覽列的「我的紀錄」按鈕。
  2. 首先檢視預設顯示的「📋 神明指示」分頁表單。
  3. 接著，點擊上方的「💰 隨喜明細」頁籤以切換分頁。
* **預期結果**：
  - 預設分頁中，可以清楚看見您方才（以及歷史所有）抽中的每一支籤詩號碼與對應的發生時間。
  - 當切換至隨喜明細時，內容清單會平滑地改變，並展示出您剛剛為帳戶所儲值的一筆 `+ NT$ 500` 的歷史紀錄。
* **實際狀態**：[ ] Pass / [ ] Fail / [ ] Pending
